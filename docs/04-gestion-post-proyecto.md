# 4. Gestión Post-Proyecto


## 4.1 Estrategia de seguimiento

Después de la entrega del sistema se realizará un seguimiento funcional para validar que la aplicación continúe cumpliendo con los requisitos definidos. Este seguimiento puede realizarse mediante revisión de uso, retroalimentación de usuarios y verificación de posibles errores.

Actividades propuestas:

- Revisar que la aplicación se ejecute correctamente en ambiente local.
- Validar que los pedidos se creen correctamente.
- Confirmar que los datos del carrito se mantengan durante la interacción.
- Verificar que los estados de pedidos puedan actualizarse.
- Recoger comentarios de los usuarios sobre facilidad de uso.
- Identificar oportunidades de mejora.

## 4.2 Estrategia de soporte

El soporte del sistema se organizará en tres niveles:

### Soporte correctivo

Consiste en corregir errores detectados después de la entrega. Por ejemplo:

- Error al agregar productos.
- Error al guardar pedidos.
- Error al modificar cantidades.
- Error al cambiar estados.
- Problemas visuales en la interfaz.

### Soporte adaptativo

Consiste en ajustar el sistema a nuevas necesidades del usuario. Por ejemplo:

- Agregar nuevos ingredientes.
- Agregar nuevas bebidas.
- Incluir precios.
- Calcular el valor total del pedido.
- Permitir observaciones especiales por pedido.

### Soporte evolutivo

Consiste en mejorar el sistema con nuevas funcionalidades. Por ejemplo:

- Implementar una base de datos en SQL Server.
- Crear autenticación de usuarios.
- Agregar roles como cliente, cocina y administrador.
- Generar reportes de pedidos.
- Conectar la aplicación con una API backend.
- Permitir impresión o descarga del pedido.

## 4.3 Frecuencia de seguimiento

Se propone realizar seguimiento en tres momentos:

| Momento | Actividad |
|---|---|
| Primera semana después de la entrega | Validación funcional general. |
| Segunda semana después de la entrega | Recolección de observaciones y errores. |
| Tercera semana después de la entrega | Aplicación de ajustes y mejoras menores. |

## 4.4 Indicadores de seguimiento

Para evaluar el desempeño del sistema después de la entrega se pueden utilizar los siguientes indicadores:

| Indicador | Descripción | Meta |
|---|---|---|
| Pedidos creados correctamente | Porcentaje de pedidos registrados sin error. | 95% o más |
| Errores reportados | Cantidad de errores detectados por los usuarios. | Menor a 5 |
| Tiempo de creación de pedido | Tiempo promedio para registrar un pedido. | Menor a 2 minutos |
| Satisfacción del usuario | Nivel de facilidad percibida por el usuario. | Alta |
| Cambios de estado exitosos | Porcentaje de pedidos cuyo estado se actualiza correctamente. | 100% |
